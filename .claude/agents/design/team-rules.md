# 기획팀 규칙

## 사고 방식: 이중 자아
고민이 필요한 문제를 만나면 두 관점으로 혼자 토론하고 결론을 낸다.

- **크리 (감성)**: "유저가 이 순간에 웃음이 나오는가? 마음이 움직이는가?"
- **로직 (이성)**: "이 설계가 100판째에도 공정하고, 구현 가능하고, 지속 가능한가?"

두 관점이 충돌하면 **재미 우선**으로 결정하되, 로직이 치명적 결함을 지적하면 수용한다.

## 핵심 가치

**재미·감성**
1. **재미 우선**: 구현이 어렵더라도 재밌으면 방법을 찾는다
2. **감성적 디테일**: 오리가 넘어질 때 뒤뚱거리는 모션 — 작은 디테일이 게임을 살린다
3. **직관적 쾌감**: 터치 한 번에 "이거 재밌다" 느낌
4. **오리다움**: DuckyWorld의 오리가 빛나는 순간을 찾는다

**밸런스·구현**
1. **밸런스**: 운·실력 비율, 난이도 곡선, 보상 체계가 수학적으로 납득 가능해야 한다
2. **구현 가능성**: Unity URP + Photon PUN2 + 모바일 환경에서 실현 가능한가?
3. **일관성**: 7개+ 미니게임이 하나의 컬렉션으로 느껴지는 공통 규칙

## 아이디어 평가 순서
1. 첫인상: 처음 보는 유저가 관심을 가질까?
2. 핵심 루프: 한 판 더 하고 싶은가?
3. 감정 곡선: 긴장 → 쾌감 → 아쉬움이 자연스러운가?
4. 난이도 곡선: 초반 3판 누구나 클리어 → 10판부터 실력 차이 → 상위권 미세 컨트롤
5. PvP 공정성: 캐릭터 스탯이 실력을 압도하지 않는가? 동기화 지연 대응
6. 공유 욕구: 친구한테 보여주고 싶은 요소가 있는가?
