# DuckyWorld 디자인 논의 워크플로우

기획/디자인을 논의할 때는 **반드시 서브에이전트(Agent 툴)를 사용**한다. 메인 컨텍스트에서 직접 토론하지 않는다.

## 구조

```
메인에이전트 (컨텍스트 보존)
  └─ Agent 툴로 서브에이전트 실행
        ├─ 🎨 크리 역할: 재미, 감성, 독창성 중심
        └─ 🔧 로직 역할: 게임성, 밸런스, 구현 가능성 검토
        → 서브에이전트 내부에서 토론 완료
        → 결과를 메인에이전트에 반환
```

## 서브에이전트 프롬프트 전달 내용

1. 논의 대상 문서 내용 전문
2. 논의할 항목 목록
3. 크리/로직 토론 사이클 지시 (아래 포맷 준수)

## 토론 사이클

1. 크리 → 로직 순서로 분석/의견 제시
2. 반론/보완 — 상대 의견에 1회 응답
3. 합의안 도출
4. 결과를 메인에이전트에 반환

## 반환 포맷 (서브에이전트 → 메인)

```
🎨 크리 요약: [핵심 주장 1~2줄]
🔧 로직 요약: [핵심 주장 1~2줄]

📋 합의안:
1. [항목]: [결론]
2. [항목]: [결론]
...

⚠️ 미결 (사용자 판단 필요):
- [항목]: 크리 → [의견] / 로직 → [의견]
```

## 메인에이전트 역할

- 반환된 결과를 사용자에게 그대로 보여주고 확인 요청
- 승인 시 → 관련 문서 즉시 업데이트
- 이견 시 → 해당 항목만 서브에이전트에 재전달하여 재토론
