# 미니게임 팀 토론 - 문제점 분석 및 신규 게임 제안

> **작성일**: 2026-03-16
> **참여자**: KREE (크리에이터), LOGI (로직)
> **목적**: 기존 7개 미니게임 문제점 파악 + 신규 5개 미니게임 도출

---

## 1단계: 기존 게임 문제점 토론

---

### 게임 01 - 스키 (Ski)

**KREE**: 터치 한 번으로 방향이 왼쪽↔오른쪽으로 토글되는 구조인데, 조작이 너무 이분법적이야. 좌/우밖에 없으니 화면 어디를 터치해도 결과가 같아서 금방 질릴 것 같아. 장애물도 나무·바위·눈사람 세 종류뿐이고 아이템도 전혀 없어서 시각적 변화가 거의 없거든.

**LOGI**: 동의해. 조작 자체는 직관적이라 좋은데, 전략 깊이가 없어. 방향이 두 개뿐이니 "얼마나 빨리 반응하는가"만 남는 게임이 되어버려. 개선 방향은 **좌 터치 / 우 터치로 조작을 분리**해서 플레이어가 의도적으로 방향을 선택하게 만드는 것. 눈사람 스턴 메커니즘은 좋은 아이디어이니 살리고, 아이템 슬롯(예: 속도 부스터, 잠깐 무적)도 추가하면 플레이 루프가 훨씬 풍부해질 것 같아.

**합의된 개선 방향**:
- 화면 좌/우 영역 터치로 조작 분리
- 아이템 1~2종 추가 (추후 기획)
- 장애물 종류 1~2개 추가 검토

---

### 게임 02 - 플래피덕 (Flappy Duck)

**KREE**: 솔직히 말하면 플래피버드 클론이잖아. 조작도 탭, 구조도 파이프 통과, 시점도 사이드뷰. 오리 캐릭터만 올려놓은 거랑 다를 게 없어. 이걸 그대로 두면 유저들이 "이거 플래피버드네" 하고 바로 닫을 것 같은데.

**LOGI**: 뼈아프지만 맞아. 차별화 포인트가 캐릭터뿐인 건 너무 약해. 다만 탭-상승 메커니즘 자체는 모바일에서 검증된 조작이라 완전 버리기엔 아깝고. 개선 방향은 **파이프를 오리 세계관에 맞는 장애물로 교체**(예: 연꽃줄기, 갈대, 낚싯줄)하고, **파이프 높이 패턴에 리듬 요소 추가**해서 페이크 오는 구간 넣는 것. 파워업 아이템(예: 물갈퀴 부스터 → 잠깐 중력 무시) 하나만 있어도 전혀 다른 게임이 돼.

**합의된 개선 방향**:
- 파이프 → 세계관 맞는 장애물로 교체 (시각적 독창성 확보)
- 패턴형 구간 추가 (리듬감 유도)
- 파워업 아이템 1종 추가

---

### 게임 03 - 달리기 (Run Duck)

**KREE**: 이 게임 제일 걱정돼. 장애물도 없고, 아이템도 없고, 그냥 연타만 하면 끝이잖아. 심지어 트랙 길이도 고정이라 매 플레이 변화가 0이야. 연타 자체는 재밌을 수 있는데, 지금은 뼈대만 있어.

**LOGI**: 더 큰 문제가 있어. **기기 성능과 터치 감도에 따라 공정성이 깨질 수 있어.** 같은 연타 횟수라도 기기마다 인식 속도가 다르고, 화면 크기에 따라 손가락 이동 거리도 달라져. 멀티플레이에선 치명적인 공정성 이슈야. 개선 방향은 **연타 횟수에 상한선(cap)을 두거나**, **연타 대신 리듬 타이밍 판정으로 보완**하는 것. 그리고 트랙에 부스터 패드나 진흙 구간 같은 지형 변화 요소를 넣어서 단순 연타 이상의 전략을 요구해야 해.

**합의된 개선 방향**:
- 기기 공정성 문제 해결을 위한 연타 인식 보정 로직 필요
- 트랙 내 지형 변화 요소 추가 (부스터/감속 구간)
- 장애물 1~2종 추가하여 회피 요소 포함

---

### 게임 04 - 허들 (Hurdle Duck)

**KREE**: 허들이랑 슬라이드런 동시에 보면... 솔직히 같은 게임 아니야? 둘 다 횡스크롤, 자동 달리기, 장애물 회피, 버틴 시간 스코어. 허들은 탭 점프, 슬라이드런은 스와이프 점프인데 체감 차이가 얼마나 날지 모르겠어.

**LOGI**: 정확히 짚었어. **04와 05는 구조적으로 80% 이상 중복이야.** 이 두 게임이 같은 게임 컬렉션에 있으면 플레이어 입장에서 "같은 거 왜 두 개야?"로 느껴질 거야. 허들만의 고유 포인트인 더블점프 메커니즘은 가치가 있으니, **허들을 완전히 다른 시점이나 맵 구조로 차별화**해야 해. 예를 들어 탑뷰 장애물 코스로 바꾸거나, 실제 허들 경기처럼 레인 선택 요소를 추가하는 방향을 고려해볼 수 있어.

**합의된 개선 방향**:
- 슬라이드런과의 중복성 해소 필수 (시점 또는 게임 구조 차별화)
- 더블점프 메커니즘은 유지하되 더블점프가 핵심이 되는 퍼즐형 구간 추가 검토
- 장기적으로는 둘 중 하나를 다른 게임으로 대체하는 방안도 고려

---

### 게임 05 - 슬라이드런 (Slide Run)

**KREE**: 허들이랑 중복 얘기는 했고, 게임 자체만 보면 스와이프 판단이 재밌는데 너무 이분법적이야. 위 스와이프면 점프, 아래 스와이프면 슬라이딩. 패턴 암기 후엔 도전감이 사라질 것 같아.

**LOGI**: 스와이프 방향 판단 게임인데 시각적 단서가 명확하면 결국 반응 속도 게임으로 귀결돼. **위험도를 높이려면 복합 장애물(예: 아래는 바닥 장애물이지만 동시에 공중에도 장애물)** 이나 **연속 입력 콤보 구간**이 필요해. 그리고 허들과의 명확한 시각적 차별화가 없으면 유저가 둘을 구분 못할 수 있어. 슬라이드런을 3D 쿼터뷰로 바꾸거나, 레인이 여러 개인 구조로 바꾸는 것도 방법이야.

**합의된 개선 방향**:
- 복합 장애물 패턴 추가로 단순 판단 이상의 요소 도입
- 허들과 시각적 차별화 (시점 변경 검토)
- 연속 입력 콤보 구간 등 고급 패턴 추가

---

### 게임 06 - 좁은길 (Narrow Path)

**KREE**: 이 게임은 진짜 독특하고 좋아. 빠름 vs 안전함 트레이드오프가 재밌는 포인트야. 근데 맵이 고정이면 두 번째 플레이부터 그냥 암기 게임이 되어버려. 그리고 조이스틱 UI가 화면 하단을 크게 차지하면 맵 가시성이 떨어져서 답답할 수 있어.

**LOGI**: 맵 암기 문제는 **절차적 생성 맵**으로 해결해야 해. 근데 좁은길을 절차 생성하면 플레이어가 앞을 예측할 수 없어서 오히려 불공평하게 느껴질 수 있어. 해결책은 **곡선 방향은 랜덤이지만 폭은 일정하게 유지**하거나, **처음엔 길을 살짝 보여주는 미니맵** 도입이야. 조이스틱 문제는 **기울기 센서(자이로스코프)를 선택지로 추가**해서 폰 자체를 기울여 조작하는 방식을 옵션으로 주는 것도 재밌어.

**합의된 개선 방향**:
- 절차적 맵 생성 도입 (단, 공정성 확보 방식 필요)
- 조이스틱 UI 최소화 또는 자이로 조작 옵션 추가 검토
- 미니맵 또는 코너 예고 시스템 추가로 답답함 해소

---

### 게임 07 - 악당피하기 (Duck Dodge)

**KREE**: PvP 아이디어가 진짜 좋아. 같은 격자 위에서 동시 플레이하면서 상대가 먼저 맞으면 이기는 심리전이 재밌을 것 같아. 그런데 4x4 격자가 너무 작지 않아? 12개 칸 중 내 오리, 상대 오리, 악당들 다 올려놓으면 피할 공간이 너무 없어서 운 게임이 될 수 있어.

**LOGI**: 공간 문제는 진짜야. 악당 2~3마리 동시 등장이면 4x4에서 선택지가 거의 없어지거든. **5x5 또는 6x6으로 격자 확대**를 고려해야 해. 그리고 PvP에서 같은 화면을 두 명이 공유할 때 **기기 방향이 서로 반대**여서 한 명은 화면을 거꾸로 봐야 하는 문제가 있어. 온라인 PvP로 전환하거나, 각자 자기 기기로 플레이하는 방식이 현실적이야. 예고 모션 아이디어는 꼭 넣어야 해 — 없으면 패턴 파악이 불가능해서 운에 의존하게 돼.

**합의된 개선 방향**:
- 격자 크기 5x5 이상으로 확대
- PvP: 같은 화면 공유 방식 → 온라인 PvP 또는 별도 기기 연결 방식으로 재설계
- 악당 예고 모션 확정 추가 (방향 예측 가능하게)
- 악당 종류 다양화 (예: 대각선으로 이동하는 악당 추가)

## 2단계: 신규 미니게임 5개 도출

### 기존 게임 조작 방식 분석 (공백 파악)

| 게임 | 조작 방식 | 시점 | 장르 |
|------|-----------|------|------|
| 스키 | 탭 (방향 토글) | 쿼터뷰 탑다운 | 장애물 회피 |
| 플래피덕 | 탭 (상승) | 사이드뷰 | 장애물 통과 |
| 달리기 | 연타 | 쿼터뷰 | 레이싱 |
| 허들 | 탭/더블탭 | 사이드뷰 | 타이밍 점프 |
| 슬라이드런 | 스와이프 ↑↓ | 사이드뷰 | 반응 판단 |
| 좁은길 | 조이스틱 | 쿼터뷰 | 정밀 조작 |
| 악당피하기 | 조이스틱 (격자) | 탑뷰 | 패턴 회피 |

**비어있는 조작/장르**: 홀드(꾹 누르기), 드래그, 정확한 타이밍 판정, 리듬, 물리 기반, 퍼즐 요소, 멀티터치

---

### 신규 게임 01 - 낚시 (Fishing Duck)

**KREE**: 오리가 낚시하는 게임 어때? 물고기가 찌 근처로 천천히 다가오는데, 딱 맞는 타이밍에 탭하면 낚고, 너무 빠르거나 늦으면 놓치는 거야. 오리 세계관에 자연스럽게 맞고, 기다리다가 순간 탭하는 긴장감이 전혀 새로운 느낌이야.

**LOGI**: 좋아. 기존 게임들이 전부 "반응 속도" 게임인데, 이 게임은 **정확한 타이밍 판정**이라는 완전히 다른 스킬을 요구해. 낚시 특성상 기다리는 시간 → 순간 집중 → 탭의 흐름이 있어서 리듬감도 생겨. 난이도 상승은 물고기가 빠르게 지나치거나, 여러 마리가 동시에 접근하거나, 가짜 물고기(탭하면 안 되는 쓰레기)가 섞이는 방식으로 자연스럽게 가능해. 구현도 어렵지 않고, 오리 세계관과 찰떡이야.

**최종 채택 ✅**

| 항목 | 내용 |
|------|------|
| 장르 | 타이밍 판정 |
| 핵심 조작 | 탭 (정확한 타이밍) |
| 시점 | 사이드뷰 또는 탑뷰 |
| 스코어 방식 | 낚은 물고기 수 / 콤보 보너스 |
| 차별점 | 반응 속도가 아닌 **정확한 타이밍 판정** — 너무 빠르거나 늦으면 실패 |
| 난이도 상승 | 물고기 통과 속도 증가, 가짜 물고기(쓰레기/낡은 장화) 추가, 다중 낚싯대 관리 |

---

### 신규 게임 02 - 풍선 오리 (Balloon Duck)

**KREE**: 오리가 풍선을 잡고 올라가는 게임인데, 화면을 꾹 누르고 있으면 빠르게 올라가고, 손을 떼면 서서히 내려와. 천장에는 가시 장애물, 바닥엔 불꽃 장애물이 있어서 홀드 시간을 조절하며 중간 지대를 유지해야 해.

**LOGI**: 플래피덕이 탭-상승이라면 이건 **홀드-상승**이라는 완전히 다른 조작 언어야. 홀드의 핵심은 연속적인 높이 조절이 가능하다는 거라서 플래피덕보다 훨씬 섬세한 컨트롤이 가능해. 위험 요소도 위/아래 양쪽에 있으니 긴장감도 두 배야. 시간이 지날수록 천장-바닥 간격이 좁아지는 방식으로 난이도 상승이 자연스러워. 구현도 단순하고 재미 포텐셜이 높아.

**최종 채택 ✅**

| 항목 | 내용 |
|------|------|
| 장르 | 정밀 높이 조절 / 생존 |
| 핵심 조작 | 홀드 (누르면 상승, 떼면 하강) |
| 시점 | 사이드뷰 (횡스크롤) |
| 스코어 방식 | 버틴 시간(초) |
| 차별점 | **홀드 조작** — 탭과 달리 연속적인 힘 조절 가능, 천장/바닥 이중 위협 |
| 난이도 상승 | 안전 구간 점점 좁아짐, 상승/하강 속도 증가, 횡으로 이동하는 장애물 추가 |

---

### 신규 게임 03 - 드럼 덕 (Drum Duck)

**KREE**: 오리가 드럼을 치는 리듬 게임! 화면 좌측/우측에 드럼이 있고, 위에서 내려오는 비트 노트에 맞춰 좌측 탭 / 우측 탭을 해야 해. 비트가 점점 빨라지고, 판정 구간이 좁아져. DuckyWorld에서 유일한 리듬 게임이 될 수 있어.

**LOGI**: 리듬 게임은 분명히 장르 공백을 채워. 근데 리듬 게임은 **음악 저작권 문제**가 있어 — 유명 곡을 쓰면 비용이 들고, 자체 제작 비트는 퀄리티 리스크가 있어. 이 부분을 어떻게 해결할지가 관건이야. 그리고 판정 시스템(Perfect/Good/Miss)을 제대로 구현하려면 오디오 레이턴시 처리가 기술적으로 까다로워. 오리 세계관에 맞는 원곡 BGM을 직접 제작하는 방향으로 간다면 충분히 가능하고, 로우폴리 오리가 드럼 치는 비주얼은 확실히 임팩트 있어.

**최종 채택 ✅** (자체 제작 BGM 전제)

| 항목 | 내용 |
|------|------|
| 장르 | 리듬 게임 |
| 핵심 조작 | 좌/우 탭 (타이밍 판정) |
| 시점 | 정면 뷰 (오리가 드럼 앞에 앉은 모습) |
| 스코어 방식 | Perfect/Good/Miss 판정 기반 콤보 스코어 |
| 차별점 | **리듬 게임** — 기존 7개 게임에 없는 완전 새 장르, 오리 세계관 자체 BGM |
| 난이도 상승 | BPM 증가, 연속 좌우 노트, 동시 탭 노트 추가 |
| 주의사항 | 자체 제작 음원 필수, 오디오 레이턴시 보정 시스템 구현 필요 |

### 신규 게임 04 - 서핑 덕 (Surf Duck)

**KREE**: 오리가 파도타기를 해! 파도가 올라올 타이밍에 맞춰 화면을 기울이거나 드래그해서 파도 위를 유지하는 게임이야. 파도가 커지면 오리가 더 높이 솟구쳐서 방향 유지가 어려워져. 기존에 없는 물리 기반 조작이 핵심이야.

**LOGI**: 물리 기반 게임은 확실히 새로운 조작 감각이야. 드래그 방향으로 파도 위 균형을 잡는 건 직관적이고, 파도 크기에 따라 난이도가 자연스럽게 올라가. 오리 세계관에도 잘 맞고 — 오리는 물 위에 있는 게 자연스럽잖아. 구현 시 파도 물리 시뮬레이션이 필요하지만, 단순화된 버전(사인파 기반 파도)으로도 충분히 재밌게 만들 수 있어. **드래그 조작**이 기존 게임과 완전히 달라서 장르 다양성에 기여해.

**최종 채택 ✅**

| 항목 | 내용 |
|------|------|
| 장르 | 물리 기반 균형 게임 |
| 핵심 조작 | 드래그 (파도 위 오리 방향/균형 조절) |
| 시점 | 사이드뷰 |
| 스코어 방식 | 버틴 시간(초) + 트릭 성공 보너스 점수 |
| 차별점 | **드래그 조작 + 물리 기반** — 파도 리듬에 맞춰 균형 잡는 완전히 새로운 게임감 |
| 난이도 상승 | 파도 높이/주기 변화, 역방향 파도 추가, 공중 장애물(갈매기) 추가 |

### 신규 게임 05 - 요리사 덕 (Chef Duck)

**KREE**: 오리 요리사가 손님에게 음식을 서빙하는 게임! 여러 테이블에 손님이 앉아 있고, 각 손님이 원하는 음식을 드래그로 끌어다 주는 거야. 손님이 많아질수록 동시 서빙이 어려워지고, 틀린 음식 서빙하면 감점. 멀티터치로 양손 동시 서빙도 가능하게!

**LOGI**: 멀티터치 드래그는 기존 7개 게임에 전혀 없는 조작 방식이야. 서빙 게임 장르는 (Diner Dash 류) 모바일에서 검증된 재미가 있어. 오리 요리사 캐릭터와 귀여운 세계관 연계도 자연스럽고. 주의할 점은 **주방 → 테이블 드래그** UI를 한눈에 파악할 수 있게 설계해야 한다는 거야. 음식 종류가 너무 많으면 복잡해지니 초기엔 3~4가지로 제한하고 점점 늘리는 방식으로 난이도 조절. 이건 개발 볼륨이 다른 게임보다 조금 더 크지만, 충분히 할 만해.

**최종 채택 ✅**

| 항목 | 내용 |
|------|------|
| 장르 | 서빙/시간 관리 |
| 핵심 조작 | 드래그 (음식 → 테이블 서빙), 멀티터치 지원 |
| 시점 | 탑뷰 |
| 스코어 방식 | 시간 내 성공 서빙 수 (실수 시 감점) |
| 차별점 | **멀티터치 드래그 + 시간 관리** — 동시 다중 입력이 요구되는 유일한 게임 |
| 난이도 상승 | 손님 수 증가, 음식 종류 증가, 까다로운 손님(특정 시간 내 서빙 안 하면 화냄) 추가 |
| 주의사항 | 음식 종류 초기 3~4가지로 제한, UI 직관성 최우선 설계 |
