# Git Commit

변경사항을 DuckyWorld 커밋 메시지 규칙에 맞춰 커밋하고 GitHub에 푸시한다.

## 커밋 메시지 형식

```
대분류. 소분류. 상세내용
```

### 대분류 (필수)

| 대분류 | 용도 |
|--------|------|
| 기획 | 기획서, GDD, 게임 디자인 문서 |
| 미니게임 | 특정 미니게임 관련 작업 |
| 캐릭터 | 캐릭터 조작, 애니메이션, 모델링 |
| 시스템 | 경제, UI, 설정, 공통 시스템 |
| 문서 | 기술 문서, 아키텍처, README |
| 기타 | 위에 해당하지 않는 작업 |

### 소분류 (선택)

변경 대상을 구체적으로 명시한다.
- 미니게임명: `퍼덕퍼덕`, `슬라이드런`
- 시스템명: `조작법`, `배경`, `경제`
- 파일/기능명: `05_slide_run`, `login_flow`

### 예시

```
캐릭터. 조작법 수정
미니게임. 퍼덕퍼덕. 배경스크롤 및 배치 조정
기획. 05_slide_run. 슬라이딩 자세 및 배경 연출 확정
시스템. 경제. 골드 획득량 재조정
기타. claude sync. 상태창 수정 [v1.0.4]
```

### 특수 규칙: Claude 설정 관련 커밋

`.claude/skills/claude/` 또는 `~/.claude/` 관련 변경 커밋 시 **반드시 버전을 메시지 끝에 포함**한다.

```
기타. claude sync. 상세내용 [v{버전}]
```

버전은 `~/.claude/settings.version` 또는 `settings/settings.version` 파일에서 확인한다.

## 워크플로우

1. **변경사항 파악**: `git status`와 `git diff --stat`으로 현재 변경 내용 확인
2. **사용자에게 질문**: "어떤 작업을 했는지" 한국어로 물어본다
3. **메시지 구성**: 사용자 답변을 위 형식으로 변환하여 제시 → 확인받기
4. **실행**:
   ```bash
   git add .
   git commit -m "확정된 메시지"
   git push
   ```
5. **결과 표시**:
   - 커밋 해시
   - 커밋 메시지
   - 변경된 파일 목록 (파일명 하나씩)

## 주의사항

- 커밋 전 반드시 사용자 확인을 거칠 것
- `.env`, 시크릿 파일 등 민감 파일이 포함되어 있으면 경고
- 푸시 실패 시 원인 분석 후 안내
