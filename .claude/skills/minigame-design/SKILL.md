# 미니게임 디자인

크리(감성/재미)와 로직(밸런스/구현) 두 관점의 토론을 통해 미니게임을 기획하고 개선한다.

## 준비

1. 논의할 게임의 `.claude/docs/duckyworld/minigames/XX_game.md`를 읽는다
   - 파일이 없으면 `_template.md` 기반으로 신규 생성
2. 기존 기획 내용과 미결 항목을 파악한다
3. 사용자가 논의하고 싶은 항목을 확인한다

## 토론 실행

Agent 툴로 서브에이전트를 생성하여 크리/로직 토론을 진행한다.

서브에이전트에게 전달할 내용:
- `.claude/agents/design/creative.md`와 `.claude/agents/design/logic.md`의 역할 지시
- 해당 게임 md 파일 전문
- 논의할 항목 목록
- 아래 토론 규칙과 반환 포맷

### 토론 규칙

```
두 역할로 토론을 진행하라:
- 🎨 크리: 재미, 감성, 독창성 중심. 유저 경험 우선.
- 🔧 로직: 게임성, 밸런스, 구현 가능성 검토.

순서:
1. 크리 → 로직 순서로 각 항목에 대해 의견 제시
2. 상대 의견에 각 1회 반응/보완
3. 합의안 도출
4. 아래 포맷으로 결과만 반환 (토론 전문 불필요)
```

### 반환 포맷

```
🎨 크리 요약: [핵심 주장 1~2줄]
🔧 로직 요약: [핵심 주장 1~2줄]

📋 합의안:
1. [항목]: [결론]
2. [항목]: [결론]

⚠️ 미결 (사용자 판단 필요):
- [항목]: 크리 → [의견] / 로직 → [의견]
```

## 결과 처리

1. 토론 결과를 사용자에게 보여주고 확인 요청
2. **승인** → 해당 `minigames/XX_game.md` 즉시 업데이트
   - 신규 게임이면 `duckyworld.md` 문서 트리에도 추가
3. **이견** → 해당 항목만 서브에이전트에 재전달하여 재토론

## 품질 체크리스트

서브에이전트에게 아래 항목도 함께 검토하도록 지시한다:

- [ ] 핵심 재미 요소가 한 문장으로 설명 가능한가?
- [ ] 난이도 곡선이 자연스러운가? (초반 쉬움 → 점진적 상승)
- [ ] 조작이 직관적인가? (터치 1~2개로 핵심 플레이 가능)
- [ ] 1분 내외 플레이 타임에 적합한가?
- [ ] 다른 미니게임과 명확한 차별점이 있는가?
- [ ] PvP 적용 시 공정한 경쟁이 가능한가?
