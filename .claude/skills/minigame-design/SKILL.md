# 미니게임 디자인 스킬

## 준비 단계

1. `docs/design_workflow.md`를 읽어 서브에이전트 운용 방식을 확인한다.
2. 논의할 게임의 `minigames/XX_game.md`를 읽는다.
   - 파일이 없으면 `minigames/_template.md`를 기반으로 신규 파일을 생성한 후 진행한다.
3. 미결 항목 또는 논의할 항목을 정리한다.

## 서브에이전트 실행

`design_workflow.md`의 서브에이전트 운용 방식에 따라 Agent 툴로 서브에이전트를 실행한다.

서브에이전트에게 전달할 내용:
- 해당 게임 md 파일 전문
- 논의할 항목 목록
- 아래 토론 사이클 및 반환 포맷 지시

### 토론 사이클 지시

```
다음 두 역할로 토론을 진행하라:
🎨 크리: 재미, 감성, 독창성 중심. 유저 경험 우선.
🔧 로직: 게임성, 밸런스, 구현 가능성 검토.

순서:
1. 크리 → 로직 순서로 각 항목에 대해 의견 제시
2. 상대 의견에 각 1회 반응/보완
3. 합의안 도출
4. 아래 포맷으로 결과 반환 (토론 전문 불필요)
```

### 반환 포맷

```
🎨 크리 요약: [핵심 주장 1~2줄]
🔧 로직 요약: [핵심 주장 1~2줄]

📋 합의안:
1. [항목]: [결론]
2. [항목]: [결론]

⚠️ 미결 (사용자 판단 필요):
- [항목]: 크리 → [의견] / 로직 → [의견]
```

## 결과 처리

- 반환된 결과를 사용자에게 보여주고 확인 요청
- 승인 시 → 해당 `minigames/XX_game.md` 즉시 업데이트
  - 신규 게임이면 `DuckyWorld_GDD.md` 미니게임 목록에도 추가
- 이견 시 → 해당 항목만 서브에이전트에 재전달하여 재토론

## 미니게임 논의 체크리스트

서브에이전트에게 아래 항목도 함께 검토하도록 지시한다:
- 핵심 재미 요소가 명확한가?
- 난이도 곡선이 자연스러운가?
- 조작감이 직관적인가?
- 다른 미니게임과 차별점이 있는가?
- 미결 항목이 있는가?
