# 미니게임 디자인

크리(감성/재미)와 로직(밸런스/구현) 두 관점의 토론을 통해 미니게임을 기획하고 개선한다.

## 준비

1. 논의할 게임의 `.claude/docs/duckyworld/minigames/XX_game.md`를 읽는다
   - 파일이 없으면 `_template.md` 기반으로 신규 생성
2. 기존 기획 내용과 미결 항목을 파악한다
3. 사용자가 논의하고 싶은 항목을 확인한다

## 실행

**촘촘이** (`.claude/agents/design/designer.md`)에 위임한다.
에이전트 역할 정의는 각 에이전트 파일 참고 — 이 스킬에서 중복 기술하지 않는다.

## 토론 반환 포맷

```
🎨 크리 요약: [핵심 주장 1~2줄]
🔧 로직 요약: [핵심 주장 1~2줄]

📋 합의안:
1. [항목]: [결론]

⚠️ 미결 (사용자 판단 필요):
- [항목]: 크리 → [의견] / 로직 → [의견]
```

## 결과 처리

1. 토론 결과를 사용자에게 보여주고 확인 요청
2. **승인** → 해당 md 파일 즉시 업데이트 + `duckyworld.md` 문서 트리 반영
3. **이견** → 해당 항목만 재토론

## 품질 체크리스트

- [ ] 핵심 재미 요소가 한 문장으로 설명 가능한가?
- [ ] 난이도 곡선이 자연스러운가?
- [ ] 조작이 직관적인가?
- [ ] 1분 내외 플레이 타임에 적합한가?
- [ ] 다른 미니게임과 명확한 차별점이 있는가?
- [ ] PvP 적용 시 공정한 경쟁이 가능한가?
