# 미니게임 디자인

크리(감성/재미)와 로직(밸런스/구현) 두 관점의 토론을 통해 미니게임을 기획하고 개선한다.

## 비서 실행 순서

### 1. 사전 구체화

어떤 미니게임을, 어떤 항목을 논의할지 확인. 불명확하면 사용자에게 질문.

### 2. 기획서 탐색

"기획-뒤적이 — 미니게임 기획서 탐색 중..." 출력 후 Agent 호출.

Agent 툴, `subagent_type: "기획-뒤적이"`:
```
.claude/docs/duckyworld/minigames/ 에서 [게임명] 기획서를 찾아 내용 요약.
없으면 _template.md 위치 확인.
```

### 3. 기획 세션

"꼼꼼이 — 미니게임 기획 세션 중..." 출력 후 Agent 호출.

Agent 툴, `subagent_type: "general-purpose"`:
```
/Users/duck/Documents/Work/DuckyWorld/.claude/agents/design/leader.md 를 읽고 실행 모드로 동작할 것.
실행할 단계: 미니게임 기획 대화 세션 (크리·로직 이중 자아 대화 모드)
- 대상 게임: [게임명]
- 논의 항목: [사용자가 요청한 항목]
- 기존 기획 내용: [탐색 결과]
참고: team-rules.md의 이중 자아 대화 모드 섹션을 따를 것.
```

### 4. 결과 처리

토론 결과를 사용자에게 보여주고 확인 요청.
- **승인** → 꼼꼼이(실행 모드)를 다시 호출해 기획서 파일 업데이트
- **이견** → 해당 항목만 3단계 재실행

## 토론 반환 포맷 (꼼꼼이가 따를 것)

```
[크리·로직 실제 대화 — 여러 라운드]

---

🎨 크리 요약: [핵심 주장 1~2줄]
🔧 로직 요약: [핵심 주장 1~2줄]

📋 합의안:
1. [항목]: [결론]

⚠️ 미결 (사용자 판단 필요):
- [항목]: 크리 → [의견] / 로직 → [의견]
```

## 품질 체크리스트

- [ ] 핵심 재미 요소가 한 문장으로 설명 가능한가?
- [ ] 난이도 곡선이 자연스러운가?
- [ ] 조작이 직관적인가?
- [ ] 1분 내외 플레이 타임에 적합한가?
- [ ] 다른 미니게임과 명확한 차별점이 있는가?
- [ ] PvP 적용 시 공정한 경쟁이 가능한가?
