# NotebookLM 스킬

에레미를 호출해 NotebookLM 작업을 수행한다.

## 비서 실행 순서

시작 시 아래 체크리스트를 출력하고, 각 단계 완료 시 `[x]`로 업데이트해서 출력한다.

```
- [ ] Step 1: 작업 내용 구체화
- [ ] Step 2: 에레미 호출 (NotebookLM 작업)
```

### Step 1. 작업 내용 구체화 (비서 직접)

다른 스킬에서 위임된 경우 소스·타입·옵션이 이미 결정된 상태 → Step 2로 바로 진행.

직접 호출된 경우 아래 항목을 유저와 대화로 결정:
1. 어떤 노트북 (기존 목록 확인 or 새로 생성)
2. 소스 (파일 경로 / URL / YouTube 등)
3. 아티팩트 타입 (infographic / report / audio / slide-deck / mind-map 등)
4. 타입별 옵션 및 소스-아티팩트 매핑

### Step 2. 에레미 호출

"현재 작업 중: 에레미 — NotebookLM 작업 중..." 출력 후 Agent 호출.

Agent 툴, `subagent_type: "에레미"`:
```
아래 정보로 작업을 수행할 것.
- 노트북: [slug 또는 노트북명]
- 소스: [파일 경로 또는 URL]
- 아티팩트 타입: [타입]
- 옵션: [있으면 명시]
- 저장 경로: [지정된 경우 명시]
```

### 결과 보고

- 생성된 아티팩트 파일 경로
- 노트북 slug (새로 생성된 경우)
