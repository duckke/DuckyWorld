# 뚝딱이 위임

개발 작업을 개발팀장 뚝딱이에게 위임한다.

## 사전 구체화 단계 (비서 필수)

뚝딱이에게 넘기기 전, 비서는 다음을 먼저 점검한다:

1. **작업 내용이 충분히 구체적인가?** 아래 항목이 불명확하면 사용자에게 질문해서 확인한다.
   - 목표: 무엇을 만들거나 고치는지
   - 기대 산출물: 어떤 파일·기능이 결과물인지
   - 제약: 건드리면 안 되는 것, 커밋 여부, 의존 조건 등
2. **불명확한 부분은 먼저 사용자에게 질문**해 구체화한 뒤 위임한다.
3. 위임 시 "뚝딱이(팀장)에게 위임합니다"라고 명시한다.

## 실행 방법

현재 대화 맥락에서 목표·기대 산출물·제약을 파악한 뒤,  
**Agent 툴을 `subagent_type: "general-purpose"`** 로 호출하고 아래 형식의 프롬프트를 전달한다.
뚝딱이(팀장)가 내부에서 작업 성격에 따라 에이전트를 선택해서 실행한다.

```
/Users/duck/Documents/Work/DuckyWorld/.claude/agents/develop/leader.md 를 읽고 해당 워크플로우를 따를 것.

목표: [한 문장]
기대 산출물: [구체적 결과물]
제약: [있으면 명시, 없으면 생략]
```

결과는 재요약 없이 그대로 사용자에게 전달한다.
