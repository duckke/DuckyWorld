# Notion 동기화 스킬

소스를 받아 NotebookLM 아티팩트를 생성하고 Notion에 업로드한다.

## 비서 실행 순서

시작 시 아래 체크리스트를 출력하고, 각 단계 완료 시 `[x]`로 업데이트해서 출력한다.

```
- [ ] Step 1: 입력 유형·아티팩트 타입 결정
- [ ] Step 2: 노트북 확인
- [ ] Step 3: 에레미 호출 (아티팩트 생성)
- [ ] Step 4: 노셔니 호출 (Notion 업로드)
```

### Step 1. 입력 유형·아티팩트 타입 결정 (비서 직접)

| 입력 | 판단 기준 | 아티팩트 기본값 |
|------|-----------|-----------------|
| 기획 문서 | `.md` 파일, "기획"·"GDD" 키워드 | 인포그래픽 |
| 개발 코드 | `.cs`·`.js` 등 코드 파일, "코드"·"구현" 키워드 | 인포그래픽 |
| 외부 링크·혼합·기타 | URL, 복수 유형 혼합, 판단 불가 | 유저에게 확인 |

직접 호출이거나 불확실하면 유저에게 타입 확인 (infographic / report / audio / slide-deck / mind-map 등).

### Step 2. 노트북 확인 (비서 직접)

`.claude/docs/notebooklm/notebooks.json` 읽어서 slug 확인:
- **있으면** → 바로 다음 단계
- **없으면** → 유저에게 확인:
  - "새로 만들까요?" → slug·제목 입력받아 진행
  - "인포그래픽 없이 올릴까요?" → Step 3 건너뛰고 Step 4로

### Step 3. 에레미 호출 (아티팩트 생성)

"현재 작업 중: 에레미 — 아티팩트 생성 중..." 출력 후 Agent 호출.

Agent 툴, `subagent_type: "에레미"`:
```
아래 정보로 아티팩트를 생성하고 지정 경로에 다운로드할 것.
- 소스: [파일 경로 또는 URL]
- 아티팩트 타입: [타입]
- 노트북 slug: [slug]
- 저장 경로: .claude/docs/notebooklm/<slug>/
```

완료 후 다운로드된 파일 경로를 확인한다.

### Step 4. 노셔니 호출 (Notion 업로드)

"현재 작업 중: 노셔니 — Notion 업로드 중..." 출력 후 Agent 호출.

Agent 툴, `subagent_type: "노셔니"`:
```
아래 정보로 Notion 페이지에 업로드할 것.
- 소스 파일: [파일 경로 또는 URL]
- 아티팩트 파일: [에레미가 다운로드한 파일 경로]
- notion_map.json을 읽어 적절한 페이지를 찾아 업로드할 것
```

### 결과 보고

- 업로드된 Notion 페이지 경로
- 생성된 아티팩트 파일 경로
