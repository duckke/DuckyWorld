# Notion 동기화 스킬

소스 유형을 파악해 적절한 오케스트레이터에게 위임한다.

## 입력 유형 판단

| 입력 유형 | 판단 기준 | 처리 방식 |
|-----------|-----------|-----------|
| 기획 문서 | `.md` 파일, "기획"·"GDD" 키워드 | `꼼꼼이` 에게 위임 |
| 개발 코드 | `.cs`·`.js` 등 코드 파일, "코드"·"구현" 키워드 | `뚝딱이` 에게 위임 |
| 외부 링크·혼합·기타 | URL, 복수 유형 혼합, 판단 불가 | 비서가 유저와 대화 후 `노셔니` 직접 호출 |

- 여러 파일이 같은 유형이면 해당 오케스트레이터에 일괄 위임
- 혼합이거나 판단이 어려우면 유저에게 확인

## 오케스트레이터 역할 (꼼꼼이·뚝딱이 공통)

위임받은 오케스트레이터는 내부에서 아래 순서로 처리한다:
1. `에레미` → 소스로 인포그래픽 생성 + `.claude/docs/notebooklm/` 다운로드
2. `노셔니` → 인포그래픽 삽입 + 본문 업로드
