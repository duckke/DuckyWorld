# Notion 동기화 스킬

## 실행 흐름

### [1단계] 입력 유형 판단

| 입력 | 판단 기준 | 아티팩트 기본값 |
|------|-----------|-----------------|
| 기획 문서 | `.md` 파일, "기획"·"GDD" 키워드 | 인포그래픽 |
| 개발 코드 | `.cs`·`.js` 등 코드 파일, "코드"·"구현" 키워드 | 인포그래픽 |
| 외부 링크·혼합·기타 | URL, 복수 유형 혼합, 판단 불가 | 유저에게 확인 |

- 여러 파일이 같은 유형이면 일괄 처리
- 혼합이거나 판단 불가 시 유저에게 확인

### [2단계] 아티팩트 타입 결정

- 기획·개발 문서: 기본값 인포그래픽
- 직접 호출 시: 유저에게 확인 (infographic / report / audio / slide-deck / mind-map 등)

### [3단계] 노트북 확인

`.claude/docs/notebooklm/notebooks.json`에서 slug 확인:
- **있으면** → 바로 다음 단계
- **없으면** → 유저에게 확인:
  - "새로 만들까요?" → slug·제목 입력받아 진행
  - "인포그래픽 없이 올릴까요?" → 에레미 없이 노셔니만 호출

### [4단계] 노셔니 호출

아래 정보를 완성된 지시서로 전달한다:
- 소스 파일 경로 (또는 URL)
- 아티팩트 타입
- 노트북 slug
- 저장 경로: `.claude/docs/notebooklm/<slug>/`

노셔니가 내부에서 에레미 호출 → 아티팩트 생성 → Notion 업로드까지 처리.
