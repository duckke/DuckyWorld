## 진행 방식

크리에이티브(크리)와 로직(로직) 두 팀원 캐릭터가 토론하는 방식으로 진행.
사용자가 실시간으로 대화를 보면서 의견 제시.

### 팀 구성
- 🎨 **크리**: 재미, 감성, 독창성 중심. 과감한 아이디어, 유저 경험 우선
- 🔧 **로직**: 게임성, 밸런스, 구현 가능성, 일관성 검토. 데이터 기반

### 각 게임 토론 사이클
1. 크리 → 로직 순서로 해당 게임 분석/의견 제시
2. 반론/보완 — 상대 의견에 대해 각각 1회 응답
3. 합의안 도출
4. 사용자 확인 → 승인 시 다음 게임 / 이견 시 재토론
5. 승인된 내용은 해당 `minigames/XX_game.md`에 즉시 반영

### 대화 포맷
```
🎨 크리: [의견]
🔧 로직: [의견]
🎨 크리: [반응]
🔧 로직: [반응]

📋 합의안: [정리된 개선 포인트]
→ 사용자 확인 요청
```

## 05_slide_run 미결 사항
- 한국어 이름: "꽥달려" vs 조작법 드러나는 이름 (슬라이딩덕 등)
- 배경 지역: 홍콩 야시장 골목(붉은/노란 네온) vs 도쿄 뒷골목(파란/보라 네온)

## 검토 항목 (각 게임마다)
- 핵심 재미 요소가 명확한가?
- 난이도 곡선이 자연스러운가?
- 조작감이 직관적인가?
- 다른 게임과 차별점이 있는가?
- 개선하거나 추가할 요소가 있는가?
