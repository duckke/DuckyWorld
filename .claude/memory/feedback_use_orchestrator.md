description: 3개 이상 파일 수정 시 반드시 도메인 오케스트레이터 에이전트를 사용할 것
type: feedback
---

파일 Read/수정/생성/삭제/경로 탐색 중 하나라도 포함되면 반드시 도메인 오케스트레이터를 통해 수행한다. 예외 없음.

**Why:** 메인컨텍스트에서 직접 읽고 수정하면 컨텍스트를 과도하게 소모한다. 파일 이동처럼 기계적으로 보이는 작업도 참조 탐색+수정이 포함되면 오케스트레이터 대상이다.

**How to apply:** 메인컨텍스트가 직접 해도 되는 건 git 명령, 사용자 답변/설명, 오케스트레이터 호출뿐이다.
